Mutable vs immutable

Immutable object often means recreating object (for example with .copy(...)). Although perfectly valid in a functional approach, creating a lot of object in the "hot path" has a high cost in the JVM performance and should be avoided.

Collections

Coming from java, scala collection can be tricky.
2 implementations exist for most of the collections: mutable and immutable.
It's important to understand the cost of accessing / adding an element to a collection. For example, accessing an element in a List has a cost in O(n).
Working with spark, ArrayBuffer are often used. Internally, spark uses a more efficient CompactBuffer for some RDD operations. https://github.com/QuentinAmbard/doc/blob/master/src/main/scala/tutorial/CollectionTutorial.scala
